## YYYY-MM-DD - Hardcoded Sentry DSN Secrets Leakage
**Vulnerability:** Hardcoded Sentry DSN URLs with sensitive credentials (`https://758053551e0396eab52314bdbcf57924@...`) were discovered in several files across the codebase (`server.js`, `generate-sitemap.js`, `sentry.ts`, and `app.py`).
**Learning:** Storing DSNs and secrets directly in source files instead of using environment variables exposes credentials, risking unauthorized telemetry access, malicious payload injection, and potential quota exhaustion, as stated in `EXTERNAL_APIS.md` and repository guidelines. Sentry initialization will fail if the DSN configuration is omitted or not safely fetched from environment variables.
**Prevention:** Remove all hardcoded DSN instances and strictly use `process.env.SENTRY_DSN`, `import.meta.env.VITE_SENTRY_DSN`, or `os.environ.get("SENTRY_DSN")`. Always implement environment checks (e.g., `if (!SENTRY_DSN)`) to ensure the application aborts initialization safely or logs a warning instead of attempting a failed connection when secrets are unconfigured. Update documentation (like `EXTERNAL_APIS.md`) to reflect `[REDACTED]` or example placeholders.
